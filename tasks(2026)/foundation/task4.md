# Task 4 - 后续

## 学习目的

学习 AI 未来有两条出路，科研和应用。

对于科研，目前，对于本校的本科生（乃至研究生）而言，AI 的科研更多是偏向于排列组合，而非开创性研究。

我的超级炫酷大模型做出来比传统的模型好几个百分点，就可以水一篇论文出来了。

更底层的东西偏向数学，例如设计新的损失函数，或者在优化器上做文章。这并不是本科生可以接触的领域。

而且更重要的是，AI 的科研除了基础阶段的学习之外，后续的学习强依赖于老师的资源。如果老师没有资源，学生也很难有机会接触到真正的科研项目。

所以，如果你想保研，那么去学 AI 的科研方向是不错的选择，校内实验室的选拔逻辑是——如果你绩点非常高，那么可以进去再学，否则需要先学 CS231n / d2l 之后，才能进去。

科研方向的后续考核，包含了这些内容。

对于 AI 的应用而言，目前尚处于基建过程中。AI 的服务化和产品化仍有较大提升空间。

所以对于本科生而言，如果你更倾向于应用方向，应该去学习更多后端知识，例如 [Go](https://github.com/west2-online/learn-go)。目前 AI 应用本质上仍是后端的一个分支，市场上的 AI 应用岗位也大多由后端直接转型而来。

> 事实上就是后端学几天大模型的调用就直接上岗了。

纯 AI 应用方向的**本科生**目前在就业市场上认可度相对较低。

所以，如果你考虑好了本科找工作，建议你同时学习后端知识，或者读研后再考虑应用方向。

这里所指的后端方向不是 Python 后端。Python 后端和前端一般只会在 Agent 的测试阶段用到，真正的 AI 应用上线仍需要 Go、Java、C++ 等后端语言。

不过无论你选择了科研还是应用，请记住：

是 AI 选择了 Python 作为主要语言，而不是 Python 选择了 AI。

另外补充的几个文件夹属于附属知识。例如 Python 后端可以用来快速在生产环境中验证 AI；还有数据分析，例如数学建模任务通常需要机器学习相关的前置知识。在具备这些前置知识后，学习这些内容只需要掌握基本思路和方法即可。
